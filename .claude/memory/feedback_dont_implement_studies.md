When the user asks for a study, comparison, or analysis document (e.g. "make an in-depth study and write it to X.md"), write the document only. Do not proceed to implement the recommended changes without explicit confirmation.

**Why:** User interrupted and corrected: "who told you to change - this is a study".

**How to apply:** After writing any analysis/study/comparison document, stop. Do not make code changes based on the study's recommendations until the user explicitly says to proceed.
